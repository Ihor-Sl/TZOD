## 1. Мета роботи

Ознайомитися з методами отримання даних з відкритих веб-API у форматі JSON, навчитися приводити дані з різних джерел до єдиної структури та об’єднувати їх в один DataFrame для подальшого аналізу.

---

## 2. Постановка задачі

Необхідно:

* отримати дані з **двох різних веб-сайтів** через API;
* обрати **одну категорію об’єктів** (у роботі — вакансії);
* привести поля з різних джерел до **спільної схеми**;
* об’єднати дані в один **DataFrame**;
* додати поле, що вказує на **джерело інформації**.

---

## 3. Джерела даних

У роботі використовувалися відкриті JSON API:

### 3.1 Remotive

* URL: https://remotive.com/api/remote-jobs
* Повертає список віддалених вакансій
* Підтримує параметр пошуку (`search`)

### 3.2 RemoteOK

* URL: https://remoteok.com/api
* Повертає JSON-фід вакансій
* Дані фільтруються на стороні клієнта

Обидва сервіси повертають дані у форматі **JSON**, що спрощує обробку в Python.

---

## 4. Спільна схема даних

Дані з різних API були приведені до єдиної структури:

| Поле       | Опис                          |
| ---------- | ----------------------------- |
| source     | Джерело (remotive / remoteok) |
| source_id  | Ідентифікатор вакансії        |
| title      | Назва посади                  |
| company    | Компанія                      |
| location   | Локація                       |
| job_type   | Тип роботи                    |
| category   | Категорія                     |
| tags       | Технології / теги             |
| salary_min | Мінімальна зарплата           |
| salary_max | Максимальна зарплата          |
| posted_at  | Дата публікації               |
| url        | Посилання на вакансію         |
